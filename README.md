# Mode Shifting — Freight Transport Emissions Analysis

This repository contains the data and code for an analysis of mode shifting in US freight trade and its implications for transport emissions.

---

## Repository Structure

```
Mode-Shifting-Aidan/
├── Scripts/
│   ├── functions.py                         # Core analysis functions (Python)
│   ├── census_scrape.r                      # Script to download trade data from Census API (R)
│   ├── thesisplots.ipynb                    # Visualizations and thesis figures
│   └── mode_shift_category_deep_dive.ipynb  # Category-level deep dive analysis
├── Aviation_Data/
│   └── T_T100I_SEGMENT_ALL_CARRIER-2.csv  # BTS T-100 international segment data (see below)
├── Ad_Valorem_Data/                  # Ad valorem tariff data (see note below)
├── Maps/                             # Natural Earth shapefiles for mapping
├── fuel_energy_info/                 # Fuel consumption and port coordinate reference data
├── Export_Data/                      # (not included — download with census_scrape.r)
└── Import_Data/                      # (not included — download with census_scrape.r)
```

Files generated at runtime (not included in repo):
- `all_flows.csv` — combined trade dataset, generated by the data pipeline below
- `all_4digits_data.xlsx` — HS4-level trade data, generated via API or placed manually
- `ocean_distances_cache.csv` — routing cache built automatically on first run of `general_emissions()`

## Setup

### Python dependencies

```bash
python -m venv .venv
source .venv/bin/activate
pip install geopandas pandas numpy matplotlib pycountry cartopy haversine \
            requests tqdm plotly seaborn statsmodels kaleido pyproj searoute
```

> `searoute` is used to compute realistic ocean routing distances. If it is not installed,
> the analysis falls back to straight-line (haversine) distances multiplied by a 1.15
> detour factor, which is still a valid approximation.

### R dependencies

The Census data download script requires R with two packages:

```r
install.packages(c("httr", "jsonlite"))
```

### Census API Key

Several functions in `Scripts/functions.py` call the US Census Bureau trade API. A free key is required:

1. Sign up at https://api.census.gov/data/key_signup.html
2. Set the key as an environment variable before running any scripts:

```bash
export CENSUS_API_KEY="your_key_here"
```

To avoid setting this every session, add it to your shell profile (`.zshrc`, `.bashrc`, etc.) or create a `.env` file in the repo root — just make sure `.env` is in your `.gitignore`.

## Data Pipeline

Run the following steps in order the first time you set up the repo. After that, you can load `all_flows.csv` directly.

**Step 1 — Download raw trade data (R)**

```bash
Rscript Scripts/census_scrape.r
```

This populates `Export_Data/` and `Import_Data/` with one CSV per HS2 commodity chapter (01–98, excluding 77 which is absent from the Census dataset). To pull a different year, open `census_scrape.r` and change the `url_end` variable in both the exports and imports sections:

```r
url_end <- "&YEAR&COMM_LVL=HS2&time=2023-12"
```

**Step 2 — Clean and combine the data (Python)**

In a notebook or Python session (from the repo root):

```python
import functions

functions.clean_all()             # cleans all raw CSV files in place — only run once on fresh downloads
all_flows = functions.concat_all()
all_flows.to_csv('all_flows.csv', index=False)
```

> Warning: `clean_all()` overwrites the raw CSV files. Only run it on freshly downloaded data.

**Step 3 — Run the analysis**

Open `Scripts/thesisplots.ipynb` or `Scripts/mode_shift_category_deep_dive.ipynb`. The preamble cells handle all path configuration automatically regardless of where Jupyter is launched from.

On first run, `general_emissions()` will compute ocean routing distances and save them to `ocean_distances_cache.csv` in the repo root. Subsequent runs load from cache and are much faster.

## Aviation Data

The `Aviation_Data/` directory contains:
- `T_T100I_SEGMENT_ALL_CARRIER-2.csv` — BTS T-100 international segment data (included for convenience)

A proprietary aviation fuel burn dataset is also required for the full analysis but cannot be included in this repo. Please contact **aghart@mit.edu** if you would like access.

To re-download the T-100 data directly from BTS:

1. Navigate to: https://www.transtats.bts.gov/Tables.asp?QO_VQ=EEE&QO_anzr=Nv4%FDPn44vr4%FDf6n6v56vp5%FD%FLS14z%FDHE%FDg4nssvp%FM-%FDNyy%FDPn44vr45&QO_fu146_anzr=Nv4%FDPn44vr45
2. Select **"International Segment (All Carriers)"**
3. Click **Download**, then filter for your year of choice
4. Select exactly the following fields before downloading:

```
DEPARTURES_PERFORMED, PAYLOAD, SEATS, PASSENGERS, FREIGHT,
UNIQUE_CARRIER, UNIQUE_CARRIER_NAME, ORIGIN_AIRPORT_ID, ORIGIN,
DEST_AIRPORT_ID, DEST, AIRCRAFT_TYPE, YEAR
```

5. Place the downloaded file in `Aviation_Data/` as `T_T100I_SEGMENT_ALL_CARRIER-2.csv`

## HS4-Level Data

The notebooks also use an HS4-level dataset (`all_4digits_data.xlsx`) for the category deep-dive analysis. This file is too large for GitHub. You have two options:

**Option A** — Place the file manually in the repo root. The notebooks will detect and load it automatically.

**Option B** — Let the notebook rebuild it from the Census API (slow, but fully reproducible). If the file is not found, `thesisplots.ipynb` will run the API rebuild automatically.
